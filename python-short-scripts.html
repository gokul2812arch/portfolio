Solving Logical problems with code (constraint based)
Someone sent me this logical problem, claiming that it was really tough and that i would not be also to solve it. And since there cant be anything that i cant solve, i was up to the task.
anyway even though it seems to be a simple puzzle, which you would think can be solved in a few minutes, it took me 15-20 minutes to actually do so. The method of solving this puzzle is simple, you start with a assumption (i.e. the location of a number) and try to figure out if the assumption was correct or not based on the other constraints. if it is wrong you move on to the next assumption until you solve it.
So after all of this i though that there should be a better way to do this, a faster way. this is where python comes in and the simple ideology of:
Why don't i test every single combination.
import time start_time = time.monotonic() def find(): for j in range(1,10): for i in range(1000,10000): d = [int(x) for x in str(i)] if len(d) == len(set(d)): check = [] check = check + d[:2] check.append(j) check = check + d[2:] if len(check) == len(set(check)): x1 = (d[0]*10 + d[1]) * j if x1 < 100 and x1 > 10: split1 = [int(x) for x in str(x1)] if split1[0] not in check and split1[1] not in check: check = check + split1 x2 = x1 + d[2]*10 + d[3] if x2 < 100 and x2 > 10: split2 = [int(x) for x in str(x2)] if split2[0] not in check and split2[1] not in check: check = check + split2 return check print(find()) print('seconds: ', time.monotonic() - start_time) # result [1, 7, 2, 5, 6, 3, 4, 9, 0] seconds: 0.045999999972991645
Python
well not the cleanest looking code, but it works. and all it took was less than a few millisecond to solve it (excluding the time taken to write the actual code) and with a nifty time module i knew exactly how long it took to run.
Well i can do better probably in fewer lines of code, why dont i use a constraint based solver to just solve the simple problem. it will probably look more refined or cleaner. and i wont have to spend time understanding the problem in essence and debug the code , which the above approach required.
Constraint based approach (used the constraint library)
from constraint import * import time start_time = time.monotonic() problem = Problem() variables = ["a","b","c","d","e","f","g","h","i"] domain = [1,2,3,4,5,6,7,8,9] # variables for i in variables: problem.addVariable(i, domain) # constraints problem.addConstraint(AllDifferentConstraint()) problem.addConstraint(lambda a, b, c, d, e: (a*10+b)*c == d*10+e,("a","b","c","d","e")) problem.addConstraint(lambda d, e, f, g, h, i: d*10+e + f*10+g == h*10+i,("d","e","f","g","h","i")) problem.addConstraint(lambda i: i<100,("i")) dict1 = problem.getSolutions()[0] #print(dict1) sortdict = {k: v for k, v in sorted(dict1.items(), key=lambda item: item[0])} print(sortdict.values()) print('seconds: ', time.monotonic() - start_time) # result dict_values([1, 7, 4, 6, 8, 2, 5, 9, 3]) seconds: 0.10999999998603016
Python
Well the number of lines did decrease , sort off , the actual code that runs is just 4 lines, the rest is just importing and setting up the variables. still its quite long. and this took longer to run a whole magnitude longer. Still need a better solution. Well what if i use no code at all
